    # keep-alive

    # keep-alive 的作用

    在Vue中，<keep-alive>组件用于对动态组件进行缓存，以便在组件之间进行快速切换时保留组件的状态，从而提高性能和用户体验。它可以将已经渲染过的组件实例保存在内存中，而不是每次切换时都销毁和重新创建组件

    KeepAlive 是一个内置抽象组件，它的功能是在多个组件间动态切换时缓存被移除的组件实例

    • 缓存组件状态：当组件被包裹在<keep-alive>中时组件的状态将会被缓存。这意味着在组件被切换出去后，组件的状态仍然被保留，包括数据、计算属性、事件监听器等
    • 节省性能开销：由于被缓存的组件不需要每次都重新创建和销毁，可以减少组件的初始化和销毁所带来的性能开销，提高页面的响应速度
    • 能在组件切换过程中将状态保留在内存中，防止重复渲染 DOM
    • 避免反复渲染影响页面性能，同时也可以很大程度上减少接口请求，减少服务器压力
    • 能够进行路由组件缓存和组件缓存
    • 提供钩子函数：<keep-alive>组件提供了一些钩子函数，可以在组件被缓存或激活时进行相应的操作。例如，可以使用activated钩子函数在组件被激活时执行一些逻辑，使用deactivated钩子函数在组件被缓存时执行一些清理操作

    # keep-alive 注册

    在初始化全局 api 执行 initGlobalAPI 时，会将内置组件 KeepAlive 注册到全局

export function initGlobalAPI(Vue: GlobalAPI) {
	...

	extend(Vue.options.components, builtInComponents)
}

    # KeepAlive 组件和普通组件不同

    • KeepAlive 组件的渲染是手写 render 函数实现的，普通组件是写模板，利用 vue 编译成 render函数
    • KeepAlive 组件不会渲染任何实体节点，是一个抽象组件

    # KeepAlive 组件的 props

    • include，包含的组件，可以是一个以英文逗号分隔的字符串、一个正则表达式，或是包含这两种类型的一个数组
    • exclude，排除的组件，可以是一个以英文逗号分隔的字符串、一个正则表达式，或是包含这两种类型的一个数组
    • max，最大缓存实例数，遵循 LRU 缓存策略

    # LRU 缓存策略

    LRU 缓存策略，即采用最近最少使用的缓存策略，它的原则是，如果一个数据最近没有被访问到，那么它将来被访问的几率也很小，也就是说当限定的内存空间已没有其他空间可用时，应该把最久没有访问到的数据去除掉。

    # KeepAlive 组件生命周期

    keep-alive 组件的渲染氛围首次渲染和缓存渲染，当命中缓存时，不会再执行 created 和 mounted 钩子函数，而会执行 activated 钩子函数。销毁时也不会执行 destroyed 钩子函数，而会执行 deactivated 钩子函数

    # keep-alive 如何刷新

    • 在 KeepAlive 组件中添加 include 或 exclude
    • 在生命周期钩子函数 activated 中执行更新操作
