    移动端适配

    # 移动端适配

    # rem适配

    rem 适配本质上是布局等比例缩放，通过设置 html 的 font-size 来改变 rem 大小

    # viewport 配置

    scale设置成固定1倍的视口大小，也可以根据dpr的值缩放viewport

    // 下面是根据设备dpr设置viewport
var dpr = window.devicePixelRatio || 1

var scale = 1 / dpr

viewport.setAttribute(
    "content",
    "width=device-width" +
    ",initial-scale=" +
     scale +
    ", maximum-scale=" +
     scale +
    ", minimum-scale=" +
     scale +
    ", user-scalable=no"

    # 设置rem基准值

    // set 1rem = 逻辑像素（设备独立像素） / 10
function setRemUnit () {
    var rem = document.documentElement.clientWidth / 10
    // 375/10 = 37.5
    docEl.style.fontSize = rem + 'px'
}
setRemUnit()

    # vw适配

    vw是基于viewport视窗(浏览器可视化区域，window.innerWidth / window.innerHeight的大小)的长度单位；1vw 等于浏览器宽度的 1%，100vw 即整个浏览器的宽度

    # 搭配vw和rem

    • 给根元素大小设置随着视口变化而变化的vw单位，这样就可以实现动态改变其大小
    • 限制根元素字体大小的最大最小值，配合body加上最大宽度和最小宽度

    代码如下：

    // rem 单位换算：定为 75px 只是方便运算，750px-75px、640-64px、1080px-108px，如此类推
$vm_fontsize: 75; // iPhone 6尺寸的根元素大小基准值
@function rem($px) {
     @return ($px / $vm_fontsize ) * 1rem;
}
// 根元素大小使用 vw 单位
$vm_design: 750;
html {
    font-size: ($vm_fontsize / ($vm_design / 2)) * 100vw; 
    // 同时，通过Media Queries 限制根元素最大最小值
    @media screen and (max-width: 320px) {
        font-size: 64px;
    }
    @media screen and (min-width: 540px) {
        font-size: 108px;
    }
}
// body 也增加最大最小宽度限制，避免默认100%宽度的 block 元素跟随 body 而过大过小
body {
    max-width: 540px;
    min-width: 320px;
}

    # px适配

    并不是说移动端就一定要使用相对长度单位，传统的响应式布局依然是很好的选择，尤其在新闻，社区等可阅读内容较多的场景直接使用px单位可以营造更好地体验。px方案可以让大屏幕手机展示出更多的内容，更符合人们的阅读习惯。

    # rem布局原理

    rem和em很容易混淆，其实两个都是css的单位，并且也都是相对单位。先介绍下em

    em作为font-size的单位时，其代表父元素的字体大小，em作为其他属性单位时，代表自身字体大小 --MDN

    em做弹性布局，需要复杂的计算并且还在于牵一发而动全身的缺点(一旦某个节点的字体大小发生变化，那么其后代元素都得重新计算)

    rem作用于非根元素时，相对于根元素(html)字体大小；rem作用于根元素字体大小时，相对于其出初始字体大小 --MDN
